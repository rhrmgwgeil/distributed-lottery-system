# 任務目標
請幫我為目前的專案全面加上「輸入驗證 (Input Validation)」與「全局錯誤處理 (Global Exception Handling)」機制。
系統必須防範不合法的資料進入 Service 層，並提供對前端友善的標準化 JSON 錯誤格式。

# 具體修改要求

## 1. DTO 層級驗證 (JSR-380 Bean Validation)
請在以下 DTO 加上對應的 validation 註解 (需 import `jakarta.validation.constraints.*`)：
- `ActivityRequestDto`:
  - `name`: `@NotBlank(message = "活動名稱不能為空")`
  - `status`: `@NotNull(message = "活動狀態不能為空")`
  - `maxDrawsPerUser`: `@NotNull`, `@Min(value = 1, message = "每人抽獎次數至少為 1")`
  - `startTime`, `endTime`: `@NotNull`
- `PrizeRequestDto`:
  - `name`: `@NotBlank(message = "獎品名稱不能為空")`
  - `stock`: `@NotNull`, `@Min(value = 0, message = "庫存不能為負數")`
  - `probability`: `@NotNull`, `@Min(value = 1, message = "機率不能小於 1"), @Max(value = 100, message = "機率不能大於 100")`
  - `prizeType`: `@NotNull`
- `UserRequestDto` & `LoginRequest`:
  - `username`: `@NotBlank(message = "帳號不能為空")`
  - `password`: `@NotBlank(message = "密碼不能為空")`

## 2. Controller 層啟動驗證
請檢視 `ActivityController`, `PrizeController`, `UserController`, `AuthController`，確保所有接收 DTO 的 API 端點，其 `@RequestBody` 前面都有加上 `@Valid` 註解。

## 3. Service 層業務邏輯驗證 (Business Rule)
- 在 `ActivityServiceImpl.java` 的 `createActivity` 與 `updateActivity` 方法最前面，加入時間邏輯校驗：
  ```java

      request.getStartTime().isAfter(request.getEndTime())) {
      throw new IllegalArgumentException("活動開始時間不能晚於結束時間");
